⏺ Your infra is well set up. Here are the steps:

  1. Install AWS SAM CLI

  brew install aws-sam-cli

  Verify with sam --version.

  2. Ensure AWS credentials are configured

  aws sts get-caller-identity

  If that fails, run aws configure or set AWS_PROFILE for the account you want
  to deploy to.

  3. Have your secrets ready

  The --guided deploy will prompt you for these parameters:

  ┌────────────────────┬─────────────────────────────────────────────────────┐
  │     Parameter      │                   What to provide                   │
  ├────────────────────┼─────────────────────────────────────────────────────┤
  │ GitHubClientId     │ Your GitHub OAuth App client ID                     │
  ├────────────────────┼─────────────────────────────────────────────────────┤
  │ GitHubClientSecret │ Your GitHub OAuth App client secret                 │
  ├────────────────────┼─────────────────────────────────────────────────────┤
  │ AppSecretKey       │ A random string for session signing (e.g. openssl   │
  │                    │ rand -hex 32)                                       │
  ├────────────────────┼─────────────────────────────────────────────────────┤
  │ ElevenLabsApiKey   │ Optional — leave blank if not using TTS             │
  └────────────────────┴─────────────────────────────────────────────────────┘

  The rest (Stage, VpcCidr, etc.) have sensible defaults in your samconfig.toml.

  4. Build and deploy

  cd infra
  sam build
  sam deploy --guided

  --guided will walk you through each parameter and save your choices to
  samconfig.toml for future deploys. After the first guided deploy, subsequent
  deploys are just sam build && sam deploy.

  5. After deploy — upload the web UI

  The stack creates an S3 bucket + CloudFront distribution for the frontend.
  You'll need to:

  # Build the frontend
  cd web && npm run build

  # Grab outputs from the stack
  sam list stack-outputs --stack-name promptdis-dev

  # Upload to the WebBucketName from outputs
  aws s3 sync dist/ s3://<WebBucketName> --delete

  # Invalidate CloudFront cache
  aws cloudfront create-invalidation --distribution-id <WebDistributionId>
  --paths "/*"

  6. Update FrontendUrl and CorsOrigins

  After the first deploy, grab the WebDistributionUrl from the stack outputs and
   redeploy with:

  sam deploy --parameter-overrides \
    Stage=dev \
    FrontendUrl=https://<cloudfront-domain> \
    CorsOrigins=https://<cloudfront-domain>

  One thing to note

  Your template sets ReservedConcurrentExecutions: 1 on the API Lambda for
  SQLite single-writer safety. This means only one Lambda invocation at a time —
   fine for dev, but you'll want to revisit that for production traffic.
